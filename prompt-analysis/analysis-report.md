## 2. Топ-5 критических улучшений

### 1. Механизм саморефлексии `[🔍 Обоснование]` (3 → 10)

**Проблема v1:** Модель не оценивает качество своих аналогий и не объясняет выбор.

**Решение v2:** На каждом шаге модель обязана написать блок обоснования:
```
[🔍 Обоснование]
1. Почему я выбрал именно это действие
2. Какую альтернативу рассмотрел и отклонил
3. Что ожидаю получить от пользователя в ответ
```

**Связь с best practice репозитория:**
> «Give Claude a way to verify its work — the single most impactful practice for quality» — Boris Cherny, Jan 2026 (tips/claude-boris-13-tips-03-jan-26.md)

Блок обоснования — это форма верификации: модель проверяет свои решения до того, как их выдаёт.

### 2. Формат вывода для диалога (4 → 10)

**Проблема v1:** 5 абстрактных заголовков без структуры полей. Монолитный формат для многоходового диалога — противоречие.

**Решение v2:** Три конкретных шаблона хода:
- **Первый ход:** Крючок → Аналогия → Объяснение → Ловушки → Граница аналогии → Вопросы
- **Ход уточнения:** Что верно → Уточнение (новый приём) → Следующий вопрос
- **Финальный ход:** Проверка → 5-компонентный снимок знания

### 3. Адаптивность и граничные случаи (4 → 9)

**Проблема v1:** Нет инструкций для нестандартных сценариев — эксперт, широкая тема, отказ отвечать.

**Решение v2:** Блок `<Edge-Cases>` с 6 сценариями + калибровка уровня при старте (4 градации вместо свободного текста).

**Связь с best practice:**
> «Build a Gotchas section — the highest-signal content in any prompt» — Thariq, Mar 2026 (tips/claude-thariq-tips-17-mar-26.md)

### 4. Измеримые ограничения (5 → 9)

**Проблема v1:** «Каждая итерация должна быть понятнее предыдущей» — неизмеримо. «Приоритет — понимание» — тавтология.

**Решение v2:** Конкретные числа и правила:
- Максимум 200 слов на блок объяснения
- Не более 2 новых терминов за ход
- Ровно 1 аналогия на объяснение
- Лексика «для 12-летнего» на начальном этапе
- Каждая итерация использует ДРУГОЙ приём (новая аналогия / смена масштаба / контрпример / мысленный эксперимент)

### 5. Адаптивная логика итераций (5 → 9)

**Проблема v1:** Жёсткий лимит «2–3 итерации» без критериев входа и выхода.

**Решение v2:**
- Диапазон: 1–5 итераций
- Критерий продолжения: пробелы в ответах пользователя
- Критерий остановки: пользователь объясняет без существенных ошибок
- Если пробелов нет после Фазы 2 → пропуск Фазы 3

## 4. Ключевая инновация v2: Самоанализ через обоснование

Главное отличие v2 — механизм **[🔍 Обоснование]**, превращающий промпт из «выполни задачу» в «выполни задачу и объясни, почему ты делаешь именно это».

Это реализует сразу три принципа:
1. **Метод Фейнмана применён к самой модели** — она объясняет свои решения, что выявляет пробелы в её логике
2. **Верификация работы** — пользователь видит рассуждения модели и может их корректировать
3. **Прозрачность** — каждый шаг обоснован, что повышает доверие к результату
